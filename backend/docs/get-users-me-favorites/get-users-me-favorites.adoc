= Метод для получения списка избранных объявлений пользователя
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет авторизованному пользователю получить постраничный список объявлений, сохранённых им в избранное. Пользователь идентифицируется по JWT-токену, переданному в заголовке `Authorization`.

В выдаче присутствуют как активные, так и скрытые администратором объявления (у скрытых `is_hidden = true`), чтобы пользователь видел, что запись помечена как скрытая, а не терял её из списка.

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization`. Если его нет или он имеет некорректный формат — вернуть исключение с http code 401 (Unauthorized) и сообщение: "отсутствует токен авторизации" / "неверный формат заголовка Authorization". Завершить работу.
. Распарсить и проверить JWT. Если токен невалиден или просрочен — вернуть исключение с http code 401 и сообщение: "невалидный или просроченный токен". Завершить работу.
. Извлечь `user_id` из claims токена и поместить его в контекст запроса. Перейти к шагу 4.2.

=== 4.2. Валидация входных параметров
|===
|Параметр |Условие |Результат

|`limit`
|Значение не является числом
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'limit' должен быть числом"

|`limit`
|Значение превышает 100
|Вернуть исключение с http code 400 и сообщение: "параметр 'limit' не может превышать 100"

|`offset`
|Значение не является числом
|Вернуть исключение с http code 400 и сообщение: "параметр 'offset' должен быть числом"

|`offset`
|Значение отрицательное
|Вернуть исключение с http code 400 и сообщение: "параметр 'offset' должен быть неотрицательным"
|===

=== 4.3. Запрос к БД на получение количества записей избранного
. Выполнить запрос `SELECT COUNT(*) FROM favorites WHERE user_id = $1` с идентификатором пользователя из шага 4.1. Сохранить результат в поле `total`.
. Перейти к шагу 4.4.

=== 4.4. Запрос к БД на получение постраничной выборки избранного
. Выполнить SQL-запрос к таблице `favorites` (алиас `f`) с соединением:
.. `JOIN listings l ON f.listing_id = l.id` — краткая карточка объявления;
.. `LEFT JOIN sources s ON l.source_id = s.id` — платформа источника;
.. подзапрос на получение URL обложки: `(SELECT photo_url FROM listing_photos WHERE listing_id = l.id AND is_cover = TRUE LIMIT 1)`.
. Условие: `WHERE f.user_id = $1`; сортировка `ORDER BY f.saved_at DESC`; пагинация `LIMIT $2 OFFSET $3`.
. Фильтр по `is_hidden` не применяется — скрытые объявления остаются в выдаче.
. Для каждой строки переписать URL обложки из внутреннего формата (`http://minio:9000/...`) в относительный путь `/minio/...`.
. Перейти к шагу 4.5.

=== 4.5. Формирование итогового ответа
. Собрать тело ответа с полями `items`, `total`, `limit`, `offset`.
. Перейти к шагу 4.6.

=== 4.6. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.5, с кодом успеха 200 OK.
. Завершить работу.
