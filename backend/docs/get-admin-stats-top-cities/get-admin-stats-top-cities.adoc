= Метод для получения топа городов по числу объявлений
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет администратору получить список городов с наибольшим количеством опубликованных объявлений. В выборку попадают только объявления, у которых указано поле `city` и которые не скрыты (`is_hidden = FALSE`).

Метод доступен только пользователям с ролью `admin`.

|query
|number
|Количество городов, которые нужно вернуть (по умолчанию `10`, максимум `100`)
|10, 25, 100
|===

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization` и распарсить JWT.
.. если токен отсутствует, невалиден или просрочен — вернуть исключение с http code 401 (Unauthorized) и сообщение: "невалидный или просроченный токен". Завершить работу;
.. иначе — извлечь `user_id` и `role` из claims, перейти к шагу 4.2.

=== 4.2. Проверка роли администратора
. Проверить, что значение `role` из JWT равно `"admin"`.
.. если роль отличается — вернуть исключение с http code 403 (Forbidden) и сообщение: "доступ запрещён: требуется роль администратора". Завершить работу;
.. иначе — перейти к шагу 4.3.

=== 4.3. Валидация входных параметров
|===
|Параметр |Условие |Результат

|`limit`
|Значение не является числом
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'limit' должен быть числом"

|`limit`
|Значение больше 100
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'limit' не может превышать 100"
|===

При отсутствии параметра `limit` или при `limit <= 0` используется значение по умолчанию `10`.

=== 4.4. Запрос к БД на получение топа городов
. Выполнить запрос:
+
[source, SQL]
....
SELECT city, COUNT(*) AS count
FROM listings
WHERE city IS NOT NULL AND is_hidden = FALSE
GROUP BY city
ORDER BY count DESC
LIMIT $1
....
. Если запрос завершился ошибкой — вернуть исключение с http code 500 (Internal Server Error). Завершить работу.
. Если подходящих записей нет — вернуть пустой массив `items: []`.
. Перейти к шагу 4.5.

=== 4.5. Формирование итогового ответа
. Собрать тело ответа в виде объекта `{ "items": [...] }`, где `items` — массив пар `{ city, count }` из результата шага 4.4.
. Перейти к шагу 4.6.

=== 4.6. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.5, с кодом успеха 200 OK.
. Завершить работу.
