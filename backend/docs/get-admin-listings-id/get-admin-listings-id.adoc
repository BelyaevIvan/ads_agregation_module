= Метод для получения карточки объявления в админ-панели
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод возвращает полную карточку объявления по его идентификатору для панели администратора. В отличие от публичного `GET /listings/{id}`, возвращает **и скрытые** (`is_hidden = TRUE`) объявления тоже. Это необходимо для того, чтобы администратор мог просматривать содержимое скрытых записей (например, перед редактированием текста или удалением фотографии).

Доступен только пользователям с ролью `admin`.

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization`. Если его нет или он имеет некорректный формат — вернуть исключение с http code 401 (Unauthorized) и соответствующее сообщение. Завершить работу.
. Распарсить и проверить JWT. Если токен невалиден или просрочен — вернуть исключение с http code 401 и сообщение: "невалидный или просроченный токен". Завершить работу.
. Извлечь из claims значения `user_id` и `role`. Перейти к шагу 4.2.

=== 4.2. Проверка роли администратора
. Если значение `role` не равно `"admin"` — вернуть исключение с http code 403 (Forbidden) и сообщение: "доступ запрещён: требуется роль администратора". Завершить работу.
. Иначе — перейти к шагу 4.3.

=== 4.3. Валидация входных параметров
|===
|Параметр |Условие |Результат

|`id`
|Значение не соответствует формату UUID
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "некорректный формат UUID"
|===

=== 4.4. Запрос к БД на получение объявления
. В таблице `listings` найти запись по идентификатору **без фильтра** `is_hidden` (ключевое отличие от публичного `/listings/{id}`):
.. если запись не найдена — вернуть исключение с http code 404 (Not Found) и сообщение: "объявление не найдено". Завершить работу;
.. иначе — перейти к шагу 4.5.

=== 4.5. Запрос к БД на получение информации об источнике
. Если у найденного объявления поле `source_id` не `null`, в таблице `sources` найти запись по этому идентификатору и подтянуть поля `platform`, `title`, `external_id`. Если запрос вернул ошибку или запись не найдена — источник в ответе будет `null`, это допустимо.
. Перейти к шагу 4.6.

=== 4.6. Запрос к БД на получение фотографий объявления
. В таблице `listing_photos` найти все записи, у которых `listing_id` соответствует идентификатору объявления, отсортировать по `sort_order` ASC.
. Для каждого фото переписать URL из внутреннего формата (`http://minio:9000/...`) в относительный путь `/minio/...`.
. Перейти к шагу 4.7.

=== 4.7. Формирование итогового ответа
. Собрать тело ответа:
.. данные объявления из таблицы `listings` (включая поле `is_hidden`);
.. объект `source` из шага 4.5 (или `null`);
.. массив `photos` из шага 4.6.
. Перейти к шагу 4.8.

=== 4.8. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.7, с кодом успеха 200 OK.
. Завершить работу.
