= Метод для получения списка источников мониторинга
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет администратору получить постраничный список всех источников мониторинга (Telegram-каналов и VK-групп), за которыми следит `monitor-service`. Для каждого источника также подсчитывается общее количество связанных с ним объявлений в БД.

Метод доступен только пользователям с ролью `admin`.

== 2. Входные параметры
|===
|Параметр |Положение |Формат |Описание |Примеры

|limit
|query
|number
|Количество записей на страницу (по умолчанию `50`, максимум `100`)
|50, 100

|offset
|query
|number
|Смещение от начала выборки (по умолчанию `0`, должен быть неотрицательным)
|0, 50, 100
|===

Требуется заголовок `Authorization: Bearer <jwt>` с валидным токеном пользователя, имеющего роль `admin`.

== 3. Выходные параметры
|===
|Параметр |Формат |Описание |Примеры

|items
|object[]
|Массив источников, отсортированный по дате добавления в порядке убывания
|—

|items[].id
|string (UUID)
|Идентификатор источника
|"9f7a1c8e-2b3d-4e5f-8a9b-0c1d2e3f4a5b"

|string
|Платформа источника
|"telegram", "vk"

|items[].external_id
|string
|ID канала/группы во внешней системе
|"-1001234567890"

|items[].title
|optional string
|Отображаемое название канала/группы
|"@sneakers_moscow"

|items[].is_active
|boolean
|Признак того, что мониторинг источника включён
|true

|items[].listings_count
|number
|Количество объявлений, распознанных из этого источника (подзапрос `SELECT COUNT(*) FROM listings WHERE source_id = s.id`)
|1243

|items[].added_at
|datetime
|Время добавления источника в систему
|"2026-01-01T00:00:00Z"

|total
|number
|Общее количество источников в БД (без учёта пагинации)
|340

|limit
|number
|Значение параметра `limit` из запроса, применённое к выборке
|50

|offset
|number
|Значение параметра `offset` из запроса, применённое к выборке
|0

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization` и распарсить JWT.
.. если токен отсутствует, невалиден или просрочен — вернуть исключение с http code 401 (Unauthorized) и сообщение: "невалидный или просроченный токен". Завершить работу;
.. иначе — извлечь `user_id` и `role` из claims, перейти к шагу 4.2.

=== 4.2. Проверка роли администратора
. Проверить, что значение `role` из JWT равно `"admin"`.
.. если роль отличается — вернуть исключение с http code 403 (Forbidden) и сообщение: "доступ запрещён: требуется роль администратора". Завершить работу;
.. иначе — перейти к шагу 4.3.

=== 4.3. Валидация входных параметров
|===
|Параметр |Условие |Результат

|`limit`
|Значение не является числом
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'limit' должен быть числом"

|`limit`
|Значение больше 100
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'limit' не может превышать 100"

|`offset`
|Значение не является числом
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'offset' должен быть числом"

|`offset`
|Отрицательное значение
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'offset' должен быть неотрицательным"
|===

При отсутствии параметра `limit` или при `limit <= 0` используется значение по умолчанию `50`. При отсутствии `offset` — `0`.

=== 4.4. Запрос к БД на получение общего количества источников
. Выполнить запрос `SELECT COUNT(*) FROM sources` для получения значения `total`.
. Если запрос завершился ошибкой — вернуть исключение с http code 500 (Internal Server Error). Завершить работу.
. Перейти к шагу 4.5.

=== 4.5. Запрос к БД на получение списка источников
. Выполнить запрос с подзапросом для подсчёта объявлений каждого источника:
+
[source, SQL]
....
SELECT s.id, s.platform, s.external_id, s.title, s.is_active, s.added_at,
       (SELECT COUNT(*) FROM listings WHERE source_id = s.id)
FROM sources s
ORDER BY s.added_at DESC
LIMIT $1 OFFSET $2
....
. Если запрос завершился ошибкой — вернуть исключение с http code 500 (Internal Server Error). Завершить работу.
. Если записей нет — вернуть пустой массив `items: []`.
. Перейти к шагу 4.6.

=== 4.6. Формирование итогового ответа
. Собрать тело ответа в формате `PaginatedResponse`:
.. `items` — массив источников из шага 4.5;
.. `total` — значение из шага 4.4;
.. `limit`, `offset` — значения из запроса.
. Перейти к шагу 4.7.

=== 4.7. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.6, с кодом успеха 200 OK.
. Завершить работу.
