1. Назначение

Метод позволяет получить постраничный список объявлений с применением полнотекстового поиска, набора фильтров (бренд, категория, город, платформа, размер, цена, состояние) и сортировки. Используется на главной странице и на странице результатов поиска.

В выдачу попадают только не скрытые объявления (is_hidden = FALSE). Скрытые записи доступны только через административный эндпоинт GET /admin/listings.

Для каждого объявления в ответе возвращается лёгкая карточка со сведениями об обложке (одна фотография) — детальная карточка вместе со списком всех фотографий доступна через GET /listings/{id}.

4. Алгоритм работы

4.1. Валидация входных параметров

Параметр Условие Результат

limit

Значение не приводится к целому числу

Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'limit' должен быть положительным числом"

limit

Значение превышает 100

Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'limit' не может превышать 100"

offset

Значение не приводится к целому числу

Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'offset' должен быть неотрицательным числом"

Значение отрицательное

Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'offset' должен быть неотрицательным"

Значение не приводится к числу с плавающей точкой

4.2. Построение условий выборки

  1. Обязательным условием добавить l.is_hidden = FALSE — скрытые объявления не возвращаются публичным эндпоинтом.

  2. Если передан q, добавить условие (l.brand ILIKE '%q%' OR l.model ILIKE '%q%' OR l.original_text ILIKE '%q%').

  3. Для массивных фильтров brand, category, city, platform добавить field = ANY($arr) (поле platform берётся из таблицы sources через LEFT JOIN).

  4. Для массивов размеров size_rus, size_eu, size_us применить оператор пересечения: field && $arr. Если include_no_size = true, условие расширяется до (field && $arr OR field IS NULL).

  5. Для ценового диапазона добавить l.price >= $min и/или l.price ⇐ $max. Если include_no_price = true, добавить OR l.price IS NULL.

  6. Если передан condition, добавить l.condition = $condition.

  7. Если include_no_city = true и фильтр по городу передан, условие расширяется до (l.city = ANY($arr) OR l.city IS NULL).

  8. Перейти к шагу 4.3.

4.3. Запрос к БД на подсчёт общего количества

  1. Выполнить запрос SELECT COUNT(*) FROM listings l LEFT JOIN sources s ON l.source_id = s.id WHERE <собранные условия> — получить значение total.

  2. В случае ошибки БД — вернуть исключение с http code 500 (Internal Server Error) и сообщение: "внутренняя ошибка сервера, попробуйте позже".

  3. Перейти к шагу 4.4.

4.4. Запрос к БД на получение страницы объявлений

  1. Выполнить запрос вида:

    SELECT l.id, l.brand, l.model, l.category, l.color, l.price, l.city, l.condition,
       l.size_rus, l.size_us, l.size_eu,
       (SELECT photo_url FROM listing_photos
         WHERE listing_id = l.id AND is_cover = TRUE LIMIT 1) AS cover_photo_url,
       s.platform, l.posted_at, l.created_at
FROM listings l
LEFT JOIN sources s ON l.source_id = s.id
WHERE <собранные условия>
ORDER BY <сортировка>
LIMIT $limit OFFSET $offset

  2. Сортировка:

    1. date_desc → ORDER BY l.posted_at DESC NULLS LAST;

    2. price_asc → ORDER BY l.price ASC NULLS LAST;

    3. price_desc → ORDER BY l.price DESC NULLS LAST.

  3. Для каждой строки переписать URL обложки из внутреннего формата (http://minio:9000/…​;) в относительный /minio/…​ для проксирования через nginx.

  4. Если массивы размеров пришли NULL, заменить их на пустые массивы ([]).

  5. Перейти к шагу 4.5.

4.5. Формирование итогового ответа

  1. Собрать тело ответа:

    1. items — результат шага 4.4 (пустой массив, если ничего не найдено);

    2. total — результат шага 4.3;

    3. limit, offset — значения, применённые к выборке.

  2. Перейти к шагу 4.6.

4.6. Отправка ответа об успехе

  1. Отправить сформированное тело с кодом успеха 200 OK.

  2. Завершить работу.
