= Метод для получения динамики объявлений по дням
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет администратору получить распределение количества новых объявлений по дням за выбранный период (7, 30 или 90 дней). Используется для построения графика динамики поступления объявлений на админ-дашборде.

Метод доступен только пользователям с ролью `admin`.

== 2. Входные параметры
|===
|Параметр |Положение |Формат |Описание |Примеры

|days
|query
|number
|Период в днях, за который возвращается статистика. Допустимые значения: `7`, `30`, `90`. Значение по умолчанию: `30`.
|7, 30, 90
|===

Требуется заголовок `Authorization: Bearer <jwt>` с валидным токеном пользователя, имеющего роль `admin`.

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization` и распарсить JWT.
.. если токен отсутствует, невалиден или просрочен — вернуть исключение с http code 401 (Unauthorized) и сообщение: "невалидный или просроченный токен". Завершить работу;
.. иначе — извлечь `user_id` и `role` из claims, перейти к шагу 4.2.

=== 4.2. Проверка роли администратора
. Проверить, что значение `role` из JWT равно `"admin"`.
.. если роль отличается — вернуть исключение с http code 403 (Forbidden) и сообщение: "доступ запрещён: требуется роль администратора". Завершить работу;
.. иначе — перейти к шагу 4.3.

=== 4.3. Валидация входных параметров
|===
|Параметр |Условие |Результат

|`days`
|Значение не является числом
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'days' должен быть числом"

|`days`
|Значение не равно одному из допустимых: 7, 30 или 90
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "параметр 'days' может быть 7, 30 или 90"
|===

При отсутствии параметра `days` используется значение по умолчанию `30`.

=== 4.4. Запрос к БД на получение статистики по дням
. Выполнить запрос:
+
[source, SQL]
....
SELECT DATE(created_at) AS date, COUNT(*) AS count
FROM listings
WHERE created_at >= NOW() - INTERVAL '$days days'
GROUP BY DATE(created_at)
ORDER BY date DESC
....
. Если запрос завершился ошибкой — вернуть исключение с http code 500 (Internal Server Error). Завершить работу.
. Если за период нет объявлений — вернуть пустой массив `items: []`.
. Перейти к шагу 4.5.

=== 4.5. Формирование итогового ответа
. Собрать тело ответа в виде объекта `{ "items": [...] }`, где `items` — массив пар `{ date, count }` из результата шага 4.4.
. Перейти к шагу 4.6.

=== 4.6. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.5, с кодом успеха 200 OK.
. Завершить работу.
