= Метод для удаления объявления из избранного
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет авторизованному пользователю удалить ранее добавленное объявление из личного списка избранного. Пользователь идентифицируется по JWT-токену, переданному в заголовке `Authorization`. Удаляется только связь `(user_id, listing_id)`; само объявление не затрагивается.

При успешном удалении возвращается `204 No Content` с пустым телом. Если запись в избранном не найдена, возвращается `404`.

|Параметр |Положение |Формат |Описание |Примеры

|Authorization
|header
|string
|JWT-токен в формате `Bearer <token>`
|"Bearer eyJhbGci..."

|listing_id
|path
|UUID

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization`. Если его нет или он имеет некорректный формат — вернуть исключение с http code 401 (Unauthorized) и соответствующее сообщение. Завершить работу.
. Распарсить и проверить JWT. Если токен невалиден или просрочен — вернуть исключение с http code 401 и сообщение: "невалидный или просроченный токен". Завершить работу.
. Извлечь `user_id` из claims токена. Перейти к шагу 4.2.

=== 4.2. Валидация входных параметров
|===
|Параметр |Условие |Результат

|`listing_id`
|Значение не соответствует формату UUID
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "некорректный формат listing_id"
|===

=== 4.3. Удаление записи из таблицы избранного
. Выполнить запрос `DELETE FROM favorites WHERE user_id = $1 AND listing_id = $2`.
. Получить количество затронутых строк (`RowsAffected`):
.. если строк удалено 0 — вернуть исключение с http code 404 (Not Found) и сообщение: "объявление не найдено в избранном". Завершить работу;
.. иначе — перейти к шагу 4.4.

=== 4.4. Отправка ответа об успехе
. Отправить пустой ответ с кодом успеха 204 (No Content).
. Завершить работу.
