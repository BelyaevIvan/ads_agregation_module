= Метод для получения карточки объявления
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет получить полную карточку объявления по его идентификатору, включая информацию об источнике и все прикреплённые фотографии.

Возвращается только если объявление **не скрыто** (`is_hidden = FALSE`). Скрытые объявления доступны только через админский эндпоинт `GET /admin/listings/{id}`.

=== 4.1. Валидация входных параметров
|===
|Параметр |Условие |Результат

|`id`
|Значение не соответствует формату UUID
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "некорректный формат UUID"
|===

=== 4.2. Запрос к БД на получение объявления
. В таблице `listings` найти запись по идентификатору, для которой `is_hidden = FALSE`:
.. если запись не найдена — вернуть исключение с http code 404 (Not Found) и сообщение: "объявление не найдено". Завершить работу;
.. иначе — перейти к шагу 4.3.

=== 4.3. Запрос к БД на получение информации об источнике
. Если у найденного объявления поле `source_id` не null, в таблице `sources` найти запись по этому идентификатору и подтянуть поля `platform`, `title`, `external_id`. Если запрос вернул ошибку или запись не найдена — источник в ответе будет `null`, это допустимо.
. Перейти к шагу 4.4.

=== 4.4. Запрос к БД на получение фотографий объявления
. В таблице `listing_photos` найти все записи, у которых `listing_id` соответствует идентификатору объявления, отсортировать по `sort_order` по возрастанию.
. Для каждого фото переписать его URL из внутреннего формата (`http://minio:9000/...`) в относительный путь `/minio/...`, чтобы браузер мог подгружать файлы через nginx-проксирование.
. Перейти к шагу 4.5.

=== 4.5. Формирование итогового ответа
. Собрать тело ответа:
.. данные объявления из таблицы `listings`;
.. объект `source` из шага 4.3 (или `null`);
.. массив `photos` из шага 4.4.
. Перейти к шагу 4.6.

=== 4.6. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.5, с кодом успеха 200 OK.
. Завершить работу.
