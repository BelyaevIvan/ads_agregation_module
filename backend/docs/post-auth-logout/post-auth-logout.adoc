== 1. Назначение
Метод предназначен для семантически корректного завершения пользовательской сессии. Поскольку в системе используется stateless-JWT без Redis/чёрного списка, реальная инвалидация токена на сервере не выполняется — токен продолжает считаться валидным вплоть до своего `exp`.

Эндпоинт проверяет наличие и валидность переданного JWT (через middleware `Auth`) и, если токен корректен, возвращает 200 OK. Фактическое "отзывание" сессии выполняется на стороне клиента: фронтенд удаляет токен из `localStorage` и очищает пользовательский контекст.

Метод требует авторизации.

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization` из запроса:
.. если заголовок отсутствует — вернуть исключение с http code 401 (Unauthorized) и сообщение: "отсутствует токен авторизации". Завершить работу;
.. если формат заголовка не соответствует шаблону `Bearer <token>` — вернуть исключение с http code 401 (Unauthorized) и сообщение: "неверный формат заголовка Authorization".
. Распарсить JWT и проверить подпись секретом сервиса. Проверить, что алгоритм подписи принадлежит семейству HMAC:
.. если алгоритм не HMAC — вернуть исключение с http code 401 (Unauthorized) и сообщение: "неподдерживаемый алгоритм подписи токена";
.. если подпись невалидна или токен истёк (`exp < now`) — вернуть исключение с http code 401 (Unauthorized) и сообщение: "невалидный или просроченный токен".
. Извлечь из клеймов `user_id`, `role`:
.. если `user_id` пустой — вернуть исключение с http code 401 (Unauthorized) и сообщение: "невалидный токен: отсутствует user_id".
. Положить `user_id` и `role` в контекст запроса.

=== 4.2. Валидация входных параметров
. Проверить, что HTTP-метод запроса — `POST`. Если нет — вернуть исключение с http code 405 (Method Not Allowed) и сообщение: "метод не поддерживается".
. Тело запроса не обрабатывается. Дополнительная валидация не требуется.

=== 4.3. Формирование итогового ответа
. Серверная инвалидация токена не выполняется: JWT stateless, чёрный список не ведётся.
. Собрать тело ответа:
+
[source, JSON]
----
{ "message": "выход выполнен успешно" }
----
. Перейти к шагу 4.4.

=== 4.4. Отправка ответа об успехе
. Отправить сформированное тело с кодом успеха 200 OK.
. Завершить работу.
