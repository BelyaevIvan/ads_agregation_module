= Метод для включения или отключения источника мониторинга
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет администратору включить или отключить мониторинг конкретного источника без его удаления. Значение поля `is_active` в таблице `sources` изменяется на указанное в теле запроса. При `is_active = FALSE` фоновый сервис `monitor-service` перестаёт опрашивать источник, но все ранее собранные объявления сохраняются в БД.

Метод доступен только пользователям с ролью `admin`.

== 2. Входные параметры
|===
|Параметр |Положение |Формат |Описание |Примеры

|id
|path
|UUID
|Идентификатор источника
|"9f7a1c8e-2b3d-4e5f-8a9b-0c1d2e3f4a5b"

|is_active
|body
|boolean
|Новое значение флага активности источника
|true, false
|===

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization` и распарсить JWT.
.. если токен отсутствует, невалиден или просрочен — вернуть исключение с http code 401 (Unauthorized) и сообщение: "невалидный или просроченный токен". Завершить работу;
.. иначе — извлечь `user_id` и `role` из claims, перейти к шагу 4.2.

=== 4.2. Проверка роли администратора
. Проверить, что значение `role` из JWT равно `"admin"`.
.. если роль отличается — вернуть исключение с http code 403 (Forbidden) и сообщение: "доступ запрещён: требуется роль администратора". Завершить работу;
.. иначе — перейти к шагу 4.3.

=== 4.3. Валидация входных параметров
|===
|Параметр |Условие |Результат

|тело запроса
|Не удалось распарсить JSON
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "некорректное тело запроса"

|`id`
|Значение не соответствует формату UUID
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "некорректный формат UUID"

|`is_active`
|Поле отсутствует в теле запроса (`null`)
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "поле 'is_active' обязательно"
|===

=== 4.4. Проверка существования источника
. Выполнить запрос `SELECT EXISTS(SELECT 1 FROM sources WHERE id = $1)`.
. Если запрос завершился ошибкой — вернуть исключение с http code 500 (Internal Server Error). Завершить работу.
. Если источник не найден — вернуть исключение с http code 404 (Not Found) и сообщение: "источник не найден". Завершить работу.
. Иначе — перейти к шагу 4.5.

=== 4.5. Обновление флага активности в БД
. Выполнить запрос:
+
[source, SQL]
....
UPDATE sources SET is_active = $2 WHERE id = $1
....
. Если запрос завершился ошибкой — вернуть исключение с http code 500 (Internal Server Error). Завершить работу.
. Перейти к шагу 4.6.

=== 4.6. Формирование итогового ответа
. Собрать тело ответа в виде `{ "id": <id>, "is_active": <новое значение> }`.
. Перейти к шагу 4.7.

=== 4.7. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.6, с кодом успеха 200 OK.
. Завершить работу.
