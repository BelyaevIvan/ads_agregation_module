= Метод для получения списка доступных размеров
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод возвращает все уникальные размеры (в сетках RUS, EU, US), встречающиеся в активных (`is_hidden = FALSE`) объявлениях. Используется фронтендом для построения чек-боксов секции «Размер» в панели фильтров поиска — вместо захардкоженного списка.

Результат кэшируется на стороне сервиса в памяти с TTL 5 минут: список размеров меняется медленно, а эндпоинт дёргается на каждом заходе в поиск, поэтому смысла бить БД на каждый запрос нет.

Метод публичный, авторизация не требуется.

== 2. Входные параметры

=== 4.1. Проверка in-memory кэша
. В сервисном слое хранится структура с тремя слайсами размеров и временем истечения кэша (`expiresAt`).
. Под read-lock проверить: есть ли сохранённый результат и актуален ли он (`now() < expiresAt`):
.. если да — вернуть копию сохранённого результата сразу. Завершить работу;
.. иначе — перейти к шагу 4.2.

=== 4.2. Параллельные запросы к БД
. Запустить три горутины, в каждой из которых выполнить свой запрос к таблице `listings`:
+
[source, SQL]
----
SELECT DISTINCT unnest(size_rus) FROM listings
WHERE is_hidden = FALSE AND size_rus IS NOT NULL;

SELECT DISTINCT unnest(size_eu) FROM listings
WHERE is_hidden = FALSE AND size_eu IS NOT NULL;

SELECT DISTINCT unnest(size_us) FROM listings
WHERE is_hidden = FALSE AND size_us IS NOT NULL;
----
. Для каждого запроса собрать выборку строковых значений, пропуская `NULL` и пустые строки.
. Если хотя бы один запрос вернул ошибку БД — вернуть исключение с http code 500 (Internal Server Error) и сообщение: "внутренняя ошибка сервера, попробуйте позже".
. Перейти к шагу 4.3.

=== 4.3. Сортировка
. Каждый из трёх списков отсортировать функцией сравнения:
.. распарсить элементы как `float64`;
.. если обе стороны — числа: сравнить численно (42 < 43, 8 < 8.5);
.. если числовой только один — его считать меньшим (нечисловые идут в конец);
.. если оба нечисловые — сравнить лексикографически.
. Перейти к шагу 4.4.

=== 4.4. Запись в кэш
. Под write-lock записать полученный результат в кэш и выставить `expiresAt = now() + 5 минут`.
. Перейти к шагу 4.5.

=== 4.5. Формирование итогового ответа
. Собрать тело ответа:
.. `size_rus` — список RUS-размеров;
.. `size_eu` — список EU-размеров;
.. `size_us` — список US-размеров.
. Перейти к шагу 4.6.

=== 4.6. Отправка ответа об успехе
. Отправить сформированное тело с кодом успеха 200 OK.
. Завершить работу.
