= Метод для добавления объявления в избранное
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет авторизованному пользователю добавить объявление в личный список избранного. Пользователь идентифицируется по JWT-токену, передаваемому в заголовке `Authorization`.

Эндпоинт **идемпотентен**: если запись уже существует (одна и та же пара `user_id` + `listing_id`), повторный вызов не создаёт дубликат и возвращает код `200 OK` с существующей записью. Если добавление произошло впервые, возвращается код `201 Created`.

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization`. Если его нет или он имеет некорректный формат — вернуть исключение с http code 401 (Unauthorized) и соответствующее сообщение. Завершить работу.
. Распарсить и проверить JWT. Если токен невалиден или просрочен — вернуть исключение с http code 401 и сообщение: "невалидный или просроченный токен". Завершить работу.
. Извлечь `user_id` из claims токена. Перейти к шагу 4.2.

=== 4.2. Валидация входных параметров
|===
|Параметр |Условие |Результат

|тело запроса
|Не является корректным JSON
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "некорректное тело запроса"

|`listing_id`
|Значение не соответствует формату UUID
|Вернуть исключение с http code 400 и сообщение: "некорректный формат listing_id"
|===

=== 4.3. Проверка существования объявления
. Выполнить запрос `SELECT EXISTS(SELECT 1 FROM listings WHERE id = $1)` с указанным `listing_id`:
.. если объявление не существует — вернуть исключение с http code 404 (Not Found) и сообщение: "объявление не найдено". Завершить работу;
.. иначе — перейти к шагу 4.4.

=== 4.4. Вставка записи в избранное
. Выполнить идемпотентный запрос:
+
[source, SQL]
----
INSERT INTO favorites (user_id, listing_id) VALUES ($1, $2)
ON CONFLICT (user_id, listing_id) DO NOTHING
RETURNING id, listing_id, saved_at
----
. Если `RETURNING` вернул строку — запись создана впервые, установить флаг "created" = true.
. Если `RETURNING` не вернул строк (произошёл конфликт) — выполнить дополнительный запрос на получение существующей записи: `SELECT id, listing_id, saved_at FROM favorites WHERE user_id = $1 AND listing_id = $2`. Установить флаг "created" = false.
. Перейти к шагу 4.5.

=== 4.5. Формирование итогового ответа
. Собрать тело ответа с полями `id`, `listing_id`, `saved_at`.
. Перейти к шагу 4.6.

=== 4.6. Отправка ответа об успехе
. Если запись создана впервые (флаг "created" = true) — отправить ответ с кодом 201 (Created).
. Иначе — отправить ответ с кодом 200 (OK), указывающим, что запись уже существовала.
. Завершить работу.
