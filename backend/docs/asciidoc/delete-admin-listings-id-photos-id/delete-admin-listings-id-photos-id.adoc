= Метод для удаления фотографии объявления
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет администратору удалить одну из фотографий объявления. Удаление выполняется как в БД (запись в таблице `listing_photos`), так и в объектном хранилище MinIO (best-effort: при ошибке в MinIO ответ клиенту остаётся успешным, так как консистентность БД приоритетнее).

Если удаляемое фото являлось обложкой (`is_cover = TRUE`), то следующая по `sort_order` фотография этого же объявления автоматически помечается как новая обложка. Её идентификатор возвращается в поле `new_cover_id` ответа. Если других фотографий не осталось либо удалялась не обложка — `new_cover_id` будет `null`.

Доступен только пользователям с ролью `admin`.

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization`. Если его нет или он имеет некорректный формат — вернуть исключение с http code 401 (Unauthorized) и соответствующее сообщение. Завершить работу.
. Распарсить и проверить JWT. Если токен невалиден или просрочен — вернуть исключение с http code 401 и сообщение: "невалидный или просроченный токен". Завершить работу.
. Извлечь из claims значения `user_id` и `role`. Перейти к шагу 4.2.

=== 4.2. Проверка роли администратора
. Если значение `role` не равно `"admin"` — вернуть исключение с http code 403 (Forbidden) и сообщение: "доступ запрещён: требуется роль администратора". Завершить работу.
. Иначе — перейти к шагу 4.3.

=== 4.3. Валидация входных параметров
|===
|Параметр |Условие |Результат

|URL
|Не соответствует шаблону `/api/v1/admin/listings/{id}/photos/{photo_id}`
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "некорректный путь запроса"

|`id`
|Значение не соответствует формату UUID
|Вернуть исключение с http code 400 и сообщение: "некорректный формат UUID"

|`photo_id`
|Значение не соответствует формату UUID
|Вернуть исключение с http code 400 и сообщение: "некорректный формат UUID"
|===

=== 4.4. Проверка существования объявления
. Выполнить запрос `SELECT EXISTS(SELECT 1 FROM listings WHERE id = $1)` с указанным `id`:
.. если объявление не существует — вернуть исключение с http code 404 (Not Found) и сообщение: "объявление не найдено". Завершить работу;
.. иначе — перейти к шагу 4.5.

=== 4.5. Получение записи фотографии
. Выполнить запрос `SELECT id, photo_url, is_cover, sort_order FROM listing_photos WHERE id = $1 AND listing_id = $2`:
.. если запись не найдена — вернуть исключение с http code 404 (Not Found) и сообщение: "фотография не найдена". Завершить работу;
.. иначе — запомнить значения полей `is_cover` (флаг `was_cover`) и `photo_url`. Перейти к шагу 4.6.

=== 4.6. Удаление записи из БД
. Выполнить запрос `DELETE FROM listing_photos WHERE id = $1` с идентификатором фото.
. Перейти к шагу 4.7.

=== 4.7. Автоматическое назначение новой обложки
. Если на шаге 4.5 флаг `was_cover` = true:
.. выполнить запрос `SELECT id FROM listing_photos WHERE listing_id = $1 ORDER BY sort_order ASC LIMIT 1`;
.. если строка найдена — выполнить `UPDATE listing_photos SET is_cover = TRUE WHERE id = $1`, сохранить `new_cover_id` = идентификатор строки;
.. если строк не осталось — `new_cover_id` = `null`.
. Если `was_cover` = false — `new_cover_id` = `null`.
. Перейти к шагу 4.8.

=== 4.8. Удаление файла из MinIO (best-effort)
. Разобрать `photo_url` (полученный на шаге 4.5), извлечь имя объекта относительно бакета. Если разбор не удался — залогировать предупреждение и пропустить удаление (без возврата ошибки клиенту).
. Вызвать `minio.RemoveObject(bucket, objectName)`. В случае ошибки MinIO — залогировать предупреждение и продолжить (ответ клиенту всё равно будет успешным: запись в БД уже удалена, а консистентность БД приоритетнее).
. Перейти к шагу 4.9.

=== 4.9. Формирование итогового ответа
. Собрать тело ответа с полями `deleted_photo_id` (идентификатор удалённой фотографии) и `new_cover_id` (идентификатор новой обложки или `null`).
. Перейти к шагу 4.10.

=== 4.10. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.9, с кодом успеха 200 OK.
. Завершить работу.
