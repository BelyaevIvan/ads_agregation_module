= Метод для редактирования оригинального текста объявления
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет администратору отредактировать оригинальный текст объявления — поле `original_text` в таблице `listings`. Это нужно для удаления нежелательной информации (контакты, спам, ненормативная лексика) без полного скрытия объявления.

Доступен только пользователям с ролью `admin`. Новый текст не может быть пустой строкой.

== 2. Входные параметры
|===
|Параметр |Положение |Формат |Описание |Примеры

|Authorization
|header
|string
|JWT-токен администратора в формате `Bearer <token>`
|"Bearer eyJhbGci..."

|id
|path
|UUID
|Идентификатор объявления
|"5a7f3c8e-1b2d-4e5f-8a9b-0c1d2e3f4a5b"

|original_text
|body (JSON)
|string (обязательное, непустое)
|Новый текст объявления. Не может быть пустой строкой
|"Продаю Nike Air Max 90 белые, 42 размер. Состояние хорошее."
|===

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization`. Если его нет или он имеет некорректный формат — вернуть исключение с http code 401 (Unauthorized) и соответствующее сообщение. Завершить работу.
. Распарсить и проверить JWT. Если токен невалиден или просрочен — вернуть исключение с http code 401 и сообщение: "невалидный или просроченный токен". Завершить работу.
. Извлечь из claims значения `user_id` и `role`. Перейти к шагу 4.2.

=== 4.2. Проверка роли администратора
. Если значение `role` не равно `"admin"` — вернуть исключение с http code 403 (Forbidden) и сообщение: "доступ запрещён: требуется роль администратора". Завершить работу.
. Иначе — перейти к шагу 4.3.

=== 4.3. Валидация входных параметров
|===
|Параметр |Условие |Результат

|тело запроса
|Не является корректным JSON
|Вернуть исключение с http code 400 (Bad Request) и сообщение: "некорректное тело запроса"

|`id`
|Значение не соответствует формату UUID
|Вернуть исключение с http code 400 и сообщение: "некорректный формат UUID"

|`original_text`
|Пустая строка
|Вернуть исключение с http code 400 и сообщение: "поле 'original_text' не может быть пустым"
|===

=== 4.4. Проверка существования объявления
. Выполнить запрос `SELECT EXISTS(SELECT 1 FROM listings WHERE id = $1)` с указанным `id`:
.. если объявление не существует — вернуть исключение с http code 404 (Not Found) и сообщение: "объявление не найдено". Завершить работу;
.. иначе — перейти к шагу 4.5.

=== 4.5. Обновление текста в БД
. Выполнить запрос `UPDATE listings SET original_text = $2 WHERE id = $1` с новым значением.
. Перейти к шагу 4.6.

=== 4.6. Формирование итогового ответа
. Собрать тело ответа с полями `id` и `original_text` (новое значение).
. Перейти к шагу 4.7.

=== 4.7. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.6, с кодом успеха 200 OK.
. Завершить работу.
