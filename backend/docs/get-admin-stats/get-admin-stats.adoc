= Метод для получения сводной статистики
:toc-title: Содержание
:table-stripes: even
:table-caption!:

== 1. Назначение
Метод позволяет администратору получить сводную статистику по системе для отображения на дашборде: общее количество объявлений (активных и скрытых), число пользователей, количество активных источников мониторинга, а также счётчики прироста за последние сутки и неделю.

Метод доступен только пользователям с ролью `admin`.

== 2. Входные параметры
Метод не принимает входных параметров. Требуется заголовок `Authorization: Bearer <jwt>` с валидным токеном пользователя, имеющего роль `admin`.

=== 4.1. Идентификация пользователя по JWT
. Извлечь заголовок `Authorization` из запроса.
.. если заголовок отсутствует или имеет неверный формат — вернуть исключение с http code 401 (Unauthorized) и сообщение: "отсутствует токен авторизации" или "неверный формат заголовка Authorization". Завершить работу;
.. иначе — распарсить JWT-токен с использованием секретного ключа. Если токен невалиден или просрочен — вернуть исключение с http code 401 (Unauthorized) и сообщение: "невалидный или просроченный токен". Завершить работу;
.. иначе — извлечь из claims поля `user_id` и `role`, перейти к шагу 4.2.

=== 4.2. Проверка роли администратора
. Проверить, что значение `role` из JWT равно `"admin"`.
.. если роль отличается от `"admin"` — вернуть исключение с http code 403 (Forbidden) и сообщение: "доступ запрещён: требуется роль администратора". Завершить работу;
.. иначе — перейти к шагу 4.3.

=== 4.3. Параллельные запросы к БД для подсчёта агрегатов
. Запустить 7 параллельных горутин, каждая выполняет отдельный запрос `COUNT(*)`:
.. `SELECT COUNT(*) FROM listings` → `total_listings`;
.. `SELECT COUNT(*) FROM listings WHERE is_hidden = FALSE` → `active_listings`;
.. `SELECT COUNT(*) FROM listings WHERE is_hidden = TRUE` → `hidden_listings`;
.. `SELECT COUNT(*) FROM users` → `total_users`;
.. `SELECT COUNT(*) FROM sources WHERE is_active = TRUE` → `active_sources`;
.. `SELECT COUNT(*) FROM listings WHERE created_at >= NOW() - INTERVAL '1 day'` → `new_listings_today`;
.. `SELECT COUNT(*) FROM users WHERE created_at >= NOW() - INTERVAL '7 days'` → `new_users_week`.
. Дождаться завершения всех горутин через `sync.WaitGroup`.
. Если хотя бы один запрос завершился ошибкой — вернуть исключение с http code 500 (Internal Server Error). Завершить работу.
. Перейти к шагу 4.4.

=== 4.4. Формирование итогового ответа
. Собрать тело ответа из 7 полученных значений в структуру `Stats`.
. Перейти к шагу 4.5.

=== 4.5. Отправка ответа об успехе
. Отправить ответ, сформированный на шаге 4.4, с кодом успеха 200 OK.
. Завершить работу.
